# Hospital Management System Design Report

## Problem Statement

Hospitals face serious challenges when appointment booking, patient records, and staff coordination are handled manually. Patients may wait in long queues just to find out whether a doctor is available. Staff may struggle with repeated paperwork, appointment conflicts, and misplaced patient information.

The proposed Hospital Management System digitizes the main patient journey. It allows users to log in, view services, select a doctor, book an appointment, receive confirmation, and access profile or medical record information.

The system is designed for patients, doctors, hospital staff, and administrators. It supports faster service delivery, better coordination, improved accessibility, and reduced waiting time.

## Final Project Choice

The chosen project idea is the **Hospital Management System**. The group should continue with this single concept for the wireframes, high-fidelity designs, interactive prototype, repository, and final documentation.

## User Flow Explanation

```text
Login/Register → Dashboard → Select Doctor → Book Appointment → Confirmation → Profile/Records
```

The user begins by logging in or creating an account. After authentication, the dashboard provides a clear overview of available hospital services. The user selects a doctor, chooses an appointment slot, confirms the booking, and can then view appointment details or medical records from the patient profile.

## Wireframe Plan

Minimum low-fidelity screens:

1. **Login/Register** — simple form with email, password, login button, and registration option.
2. **Dashboard** — overview cards for appointment booking, records, doctors, and hospital services.
3. **Doctor List / Appointment Booking** — doctor cards, specialty labels, availability, and booking button.
4. **Patient Profile / Records** — patient details, recent visits, prescriptions, and lab results.

Wireframes should use simple boxes, placeholder text, and arrows to show layout and navigation before visual styling is added.

## High-Fidelity Design Overview

Minimum polished screens:

1. Login
2. Dashboard
3. Doctor List
4. Appointment Booking
5. Appointment Confirmation
6. Patient Profile

The high-fidelity design uses a hospital color palette of blue, green, and white. Blue communicates trust and professionalism, green communicates health and recovery, and white keeps the interface clean and readable. Icons are used to support quick recognition of services such as appointments, doctors, records, and patient profiles.

Typography should remain consistent across all screens, with clear headings, readable body text, and obvious button labels. Layouts should be spacious and mobile-friendly so that patients can comfortably use the system on a phone.

## Interactive Prototype

The prototype should be linked in Figma Prototype mode using the following flow:

1. Login button opens the Dashboard.
2. Book Appointment card opens the Doctor List.
3. Doctor card opens Appointment Booking.
4. Confirm Appointment button opens Appointment Confirmation.
5. View Records button opens Patient Profile.

Transitions should include simple fade and slide animations. The prototype should be tested from login to appointment confirmation to ensure all navigation links work correctly.

## Accessibility Notes

- Font sizes should be at least 16px for readability.
- Text and buttons should maintain strong contrast against backgrounds.
- Navigation menus should be clearly labeled.
- Buttons should use consistent shapes, colors, and spacing.
- Touch targets should be large enough for mobile users.
- Important information should not rely on color alone; labels and icons should support meaning.

## Challenges Faced

Potential challenges include keeping the booking flow simple, organizing medical information without overwhelming the user, and ensuring the design works well on mobile screens. Another challenge is balancing a professional hospital look with a friendly patient experience.

## Conclusion

The Hospital Management System prototype provides a clear digital solution for appointment booking, doctor selection, confirmation, and patient record access. It addresses long queues, manual record problems, and poor scheduling visibility. With accessible design, consistent navigation, and a focused user flow, the prototype supports a more efficient and patient-friendly hospital experience.
