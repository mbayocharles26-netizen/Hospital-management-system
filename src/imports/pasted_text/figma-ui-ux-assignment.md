Objective 
The purpose of this assignment is to help students design modern, user-friendly, and interactive mobile or 
web application interfaces using Figma. 
Students will practice: 
• UI/UX design principles 
• Wireframing 
• Interface design 
• Prototyping 
• Responsive and accessible design concepts 
This assignment focuses on creating visually appealing and user-centered e-commerce or web application 
solutions. 
Group Activity 
• Students must work in groups of two (2). 
• Each group will collaboratively design one application prototype. 
• Both students must actively participate in: 
o Research 
o Design 
o Wireframing

o Prototyping 
o Documentation 
Task Description 
Each group must design a complete application prototype using Figma. 
Choose ONE System 
Groups may choose one of the following systems: 
• E-learning application 
• Online shopping application 
• Hospital appointment system 
• Food delivery application 
• Banking or digital wallet application 
• School management system 
• Event booking system 
• Fitness tracking application 
• Any other approved web/mobile application idea 
Deliverables 
1. Problem Statement 
Clearly explain: 
• What problem does the application solve? 
• Who are the target users? 
• Why is the system important? 
2. User Persona (At Least One) 
Create at least one user persona including: 
• Name 
• Age 
• Occupation 
• Goals 
• Challenges or frustrations 
3. User Flow Diagram 
Page 2 of 7 
Show the sequence of steps users follow in the system. 
Example 
Login → Home Page → Select Product → Checkout → Payment Confirmation 
The flow should clearly demonstrate navigation within the application. 
4. Wireframes (Low-Fidelity Design) 
Create simple layout sketches of the application. 
Requirements 
• Minimum of 4 screens 
• Focus on structure and layout 
• Use placeholders where necessary 
Example Screens 
• Login/Register 
• Home Page 
• Product or Service Page 
• Profile Page 
5. High-Fidelity UI Design 
Design polished and visually appealing interfaces. 
Requirements 
• Minimum of 6 screens 
• Use: 
o Colors 
o Typography 
o Icons 
o Images 
o Proper spacing and alignment 
• Maintain consistency across all screens 
6. Interactive Prototype 
Create clickable interactions in Figma. 
The prototype should: 
Page 3 of 7 
• Connect screens properly 
• Demonstrate user navigation 
• Simulate real application usage 
Examples 
• Button interactions 
• Page transitions 
• Navigation menus 
7. Accessibility Considerations 
Ensure the application is easy to use for different users. 
Include: 
• Readable font sizes 
• Good color contrast 
• Clear navigation 
• User-friendly layouts 
• Consistent design elements 
Design Requirements 
Groups must: 
• Use original design ideas 
• Create responsive/mobile-friendly layouts 
• Maintain consistency in the interface 
• Provide clear feedback to users through the interface 
• Apply modern UI/UX practices 
Recommended Figma Features 
Students are encouraged to use: 
• Frames 
• Components 
• Auto-layout 
• Prototyping tools 
• Icons and design systems 
Page 4 of 7 
• Grids and alignment tools
